MCP Host Engine Documentation

The MCP Host Engine facilitates communication between LLMs and external tools via JSON-RPC 2.0.

Modern API (Namespaced)

Plugins should utilize the following structured objects for accessing host capabilities.

1. File System (fs)

Provides robust interaction with the local filesystem.

MethodDescription
fs.readFile(path)Reads file content as string.
fs.writeFile(path, content)Writes/overwrites file.
fs.listFiles(path)Lists directory contents.
fs.deleteFile(path)Removes file or directory.
fs.renameFile(old, new)Renames or moves paths.
fs.isDir(path)Checks if path is a directory.
fs.exists(path)Checks if path exists.

2. Network (http)

Standardized HTTP operations.

MethodDescription
http.get(url, headers)Performs HTTP GET.
http.post(url, headers, body)Performs HTTP POST.
http.put(url, headers, body)Performs HTTP PUT.
http.delete(url, headers)Performs HTTP DELETE.
http.options(url, headers)Performs HTTP OPTIONS request.
http.head(url, headers)Performs HTTP HEAD request.

3. Process & System Utilities

  • exec.runCommand(command): Executes system shell commands.
  • process.env(key): Retrieves system environment variables.
  • mcp.logger: Accesses the plugin-specific logger.
  • server.config(name): Retrieves configuration for the specified plugin.

4. Utils (utils)

Provides various utility functions to JavaScript plugins

  • utils.btoa(str): Encodes a string to base64.
  • utils.atob(str): Decodes a base64 string to string.

5. URL (url)

Provides Web-compatible URL and URLSearchParams API to JavaScript plugins

MethodDescription
url.parse(url)Parses a URL.
url.searchParams(query)Creates a URLSearchParams object.
url.resolve(base, relative)Resolves a relative URL to a base URL.

Legacy API

These methods are maintained for backward compatibility and are subject to future deprecation.

readFile, writeFile, listFiles, deleteFile, renameFile, runCommand, httpGet, httpPost, httpPut, httpDelete, getEnv, readStream, writeStream, makeDir, rmDir, isdir, isexisted.
