MCP Host Engine Documentation Go Server JavaScript Plugins

This document provides detailed information about the MCP Host Engine, a Go-based server that runs tools as JavaScript plugins. This design allows you to create custom MCP servers by combining the lightweight Go host with specialized JavaScript plugins.

Current Host: MCPHE-Server

Overview

The MCP Host Engine acts as a bridge between Large Language Models (LLMs) and external tools. It uses the JSON-RPC 2.0 protocol to communicate, enabling AI agents to interact with their environment in a structured way.

Key Features

  • http or stdio Transport protocol for communication
  • Go-based Performance-optimized core
  • JavaScript Plugins Flexible and easy-to-develop tools
  • JSON-RPC 2.0 Standardized communication protocol
  • Configurable Per-server configuration files
  • Secure Built-in authentication and domain whitelisting

Getting Started

Prerequisites

  • Go 1.18+
  • Node.js (for plugin development)

Installation

To get started, clone the repository and build the server:

git clone https://github.com/bear-bearbear/mcphe
cd mcphe
go mod tidy
make build

Running the Server

Start the server with a configuration file:

./mcphe config.yaml

Testing

You can test the server using curl:

curl -X POST http://localhost:8001/rpc \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-token" \
  -d '{
    "jsonrpc": "2.0",
    "method": "mcp.listTools",
    "params": {},
    "id": 1
  }'

Plugin Development

Plugins are the core of the MCP Host Engine's flexibility. You can write custom tools in JavaScript to extend the server's capabilities.

Plugin Structure

A plugin is a JavaScript file that exports an object with the following properties:

module.exports = {
  name: "plugin-name",
  description: "Description of what this plugin does.",
  inputSchema: { /* JSON Schema */ },
  call: function(params, mcphe, tools) { /* Plugin logic */ }
}

Available Host Object Methods

The plugin logic resides within a JavaScript object (`mcphe` or `tools`) provided by the host. This object exposes methods implemented by the core Go server, allowing plugins to interact with the operating system, file system, and network.

1. File System Operations

Methods for interacting with the local file system.

readFile

Reads the contents of a file given its absolute path.

string readFile(path: string)

Returns: The file content (string).

writeFile

Writes content to a file, overwriting existing content.

error writeFile(path: string, content: string)

Parameters: path (string), content (string).

listFiles

Lists all files and directories within a specified path.

string[] listFiles(path: string)

Parameters: path (string).

deleteFile

Deletes a specified file or directory from the system.

error deleteFile(path: string)

Parameters: path (string).

renameFile

Renames or moves a file/directory from an old path to a new path.

error renameFile(oldPath: string, newPath: string)

Parameters: oldPath (string), newPath (string).

2. System and Network Operations

Methods for executing shell commands, accessing environment variables, and making HTTP requests.

runCommand

Executes an arbitrary system shell command and captures its standard output.

tuple runCommand(command: string)

Parameters: command (string). Returns a tuple of (stdout, error).

getEnv

Retrieves the value of a specified system environment variable.

tuple getEnv(key: string)

Parameters: key (string).

httpGet

Performs an HTTP GET request to a specified URL.

tuple httpGet(urlStr: string, headers: map)

Parameters: urlStr (string), headers (map).

httpPost

Performs an HTTP POST request, typically used to send structured data.

tuple httpPost(urlStr: string, headers: map, body: string)

Parameters: urlStr (string), headers (map), body (string).

httpPut

Performs an HTTP PUT request, typically used for full resource updates.

tuple httpPut(urlStr: string, headers: map, body: string)

Parameters: urlStr (string), headers (map), body (string).

httpDelete

Performs an HTTP DELETE request.

tuple httpDelete(urlStr: string, headers: map)

Parameters: urlStr (string), headers (map).

3. System Configuration

config

Retrieves a read-only object containing the runtime configuration settings for the MCP Host Engine instance.

object config()

Returns: A configuration object containing global server settings.

Method Name Description Parameters & Return Type
readFile Reads file content from the local file system. (path: string) → string
writeFile Writes or overwrites content in a specified file. (path: string, content: string) → error
runCommand Executes a shell command on the underlying host OS. (command: string) → tuple<string, error>
httpGet Performs an HTTP GET request to a remote endpoint. (url: string, headers: map) → tuple<object, error>
getEnv Retrieves the value of an environment variable by key. (key: string) → tuple<string, error>
config Accesses the immutable server configuration. () → object
